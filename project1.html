IV. Computer program needs to be structured.

That means there are some rules when writing codes. The computer reads the lines from the top to the bottom. So the logical flow of the code have to be aligned in the same direction. The code's have to be divided into meaningful components. And we need to minimize repeating same lines. If we know the structure, we can predict the organization of code's content. Well structured programs are easily read and understood.

There are several benefits of avoiding repetition. First, we can minimize the total length of the code. Therefore, there are less to write, less risks of making errors and less to read. Second, it makes future changes easier and simple by putting same components together. If we have written the same codes several times, we should have find where the same parts are and change all of them one after one.
